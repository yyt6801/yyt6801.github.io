# Git工作流与团队协作


## 集中式工作流
- 适合小团队
- 直接在main分支开发

## Feature Branch工作流
- 功能分支命名规范
- Pull Request流程

## Git Flow
- master、develop、feature、release、hotfix分支
- 各分支职责与生命周期
- 适合有版本发布节奏的项目

## GitHub Flow
- 简洁：main &#43; feature分支
- 部署即发布
- 适合持续交付场景

## 工业项目中的Git实践
- 配置文件管理策略
- 大文件管理（Git LFS）
- 多仓库管理（submodule、monorepo）
